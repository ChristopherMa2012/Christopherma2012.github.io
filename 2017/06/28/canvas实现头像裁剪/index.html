canvas实现头像裁剪

2017-06-28 • 标签: javascript canvas

公司的项目中有一个需求，就是用户可以自主选择、裁剪自己的账号头像。本着自己能完成的事情就不麻烦别人的原则，再结合前端也可以使用canvas来实现图片的操作，故有了下面的尝试,同时总结记录几个要点。

一、因为canvas的getContext() 的是原生DOM对象才有的方法，所以使用JQ的时候要注意将对象转化为原生js对象。

二、canvas在IE10上存在安全限制

除此之外，在google浏览器上使用canvas还存在图片跨域的问题。谷歌浏览器不允许使用与当前页domain不一致的图片。而在公司的项目里，www.xiaohuangdou.com和图片服务器1.img.xiaohuangdou.com是不同的两个域。因此要使用canvas还必须在图片服务器的配置里加上响应头：Access-Control-Allow-Origin:http://www.xiaohuangdou.com。

主要的技术点举例完后，如何实现QQ头像那样的图片裁剪功能呢？
首先QQ的截取面板是由一个中间为透明无色正方形，而其他区域透明但灰暗的遮罩层和底部的一张图片组成。拖动中间的那个透明无色正方形，可以选取我们想要裁剪的区域。
其次在拖动的同时，在旁边的预览窗口能够实时的显示当前透明无色正方形选取的区域。
最后，用户在确定自己想要截取的区域后，按下确认键，便能将选中的区域当作一张新的图片传送的后台存储。

由此，我们可以从上面的要点找到对应的解决方法以及需要的注意的细节处理。
第一步，先处理截取面板。直观的看，截取面板有三部分：无色透明正方形、暗色透明区域和底部图片。因为通过 var ctx = canvas.getContext(“2d”) ( 为了便于描述，以下均已ctx代替 canvas.getContext(“2d”)对象 ) 的 globalAlpha 属性来设置 图形的透明度，所以无色透明
和暗色透明的区域都有了解决办法。但是使用三个canvas实现这三个部分显得有点不符合实际，毕竟如果透明正方形和暗色透明区域用不同的canvas实现的话，是没有办法实现像QQ那种拖动无色透明正方形的同时，暗色透明区域会自动填补无色透明正方形移动的空间的。
恰好ctx对象还有一个 globalCompositeOperation 属性。globalCompositeOperation 属性设
置要在同一个canvas面板绘制新形状时应用的合成操作的类型，用大白话说就是，在一个canvas面
板上，下一步绘制的图形是如何协调在上一步绘制的图形的位置关系的。通过这个属性，我们可以将
无色透明正方形、暗色透明区域都集合在一个canvas面板上，节省了一个canvas标签。
