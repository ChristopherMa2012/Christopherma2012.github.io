ES6笔记之四‘Module’

ES6模块的设计思想是尽量静态化，使得编译时就能确定模块的依赖关系，以及输入和输出的变量。
而CommonJs和AMD模块都只能在运行时确定。
CommonJs模块就是对象，输入时必须查找对象属性 

let { stat , exists, readfile } = require('fs')  
//“运行时加载”
//整体加载fs模块(即加载fs的所有方法),而使用时只用到上面列出的三个方法。

ES6模块不是对象，通过export命令显式指定输出的代码，输入时也采用静态命令的形式 

import {stat , exists, readfile } form 'fs';
//“编译时加载”
//从fs模块加载3个指定方法，其他不加载

ES6在编译时就完成模块编译，效率比CommonJS模块加载方式高
ES6模块自动采用严格模式，不管有没有在模块头部加上“use strict”

export

一个模块就是一个独立的文件，该文件内部的所有变量外部无法获取，如果希望外部能
够读取模块内部的某个变量或者函数，必须使用export关键字输出该变量或函数 。示例： 

export var firstName = 'Michael';
//导出变量
export function func(){};  
//导出函数
function func1(){}
function func2(){}
export {func1,func2} ;
export {        //设置别名
  func1 as function1,
  func2 as function2
}
//另一种函数导出方式

export命令可以出现在模块的任何位置，只要处于模块顶层即可

import

import命令具有提升效果，会提升到模块的头部优先执行。
import接受一个对象”{}” ,该对象指定要从其他模块导入的变量名,并且必须和其他模块暴露出来的变量名一致

//profile.js
export {firstName, year}
//使用profile.js模块的文件
import { firstName , year } from './profile';
import { firstName as fn , year as y } from './profile';  
//给导入的目标设置别名

如果在一个模块中，先导入在输出同一个模块，import语句可以与export语句写在一起 

export {es6 as default } from  './module' 
//等价于
import {es6 } form './module'
export default es6

模块的整体加载

用星号*指代一个对象，所有输出值都加载在这个对象上 

//module1.js
export var firstName = 'Michael';
export function func(){};  
//module2.js
import * as obj from './module1';
obj.func();
//调用module1的方法func()
obj.firstName
//'Michael'

module命令

module obj from './module1'
//等价于
import * as obj from './module1';

module后面跟一个变量，表示输入的模块定义在该变量上

export default

为了可以使用户不用查看模块源码去寻找模块暴露的接口，使用export default为模块指定默认输出
