ES6笔记之四‘Module’

ES6模块的设计思想是尽量静态化，使得编译时就能确定模块的依赖关系，以及输入和输出的变量。
而CommonJs和AMD模块都只能在运行时确定。
CommonJs模块就是对象，输入时必须查找对象属性 

let { stat , exists, readfile } = require('fs')  
//“运行时加载”
//整体加载fs模块(即加载fs的所有方法),而使用时只用到上面列出的三个方法。

ES6模块不是对象，通过export命令显式指定输出的代码，输入时也采用静态命令的形式 

import {stat , exists, readfile } form 'fs';
//“编译时加载”
//从fs模块加载3个指定方法，其他不加载

ES6在编译时就完成模块编译，效率比CommonJS模块加载方式高
ES6模块自动采用严格模式，不管有没有在模块头部加上“use strict”

export

一个模块就是一个独立的文件，该文件内部的所有变量外部无法获取，如果希望外部能
够读取模块内部的某个变量或者函数，必须使用export关键字输出该变量或函数 。示例： 

export var firstName = 'Michael';
//导出变量
export function func(){};  
//导出函数
function func1(){}
function func2(){}
export {func1,func2} ;
export {        //设置别名
  func1 as function1,
  func2 as function2
}
//另一种函数导出方式

export命令可以出现在模块的任何位置，只要处于模块顶层即可

import

import命令具有提升效果，会提升到模块的头部优先执行。
import接受一个对象”{}” ,该对象指定要从其他模块导入的变量名,并且必须和其他模块暴露出来的变量名一致

//profile.js
export {firstName, year}
//使用profile.js模块的文件
import { firstName , year } from './profile';
import { firstName as fn , year as y } from './profile';  
//给导入的目标设置别名

如果在一个模块中，先导入在输出同一个模块，import语句可以与export语句写在一起 

export {es6 as default } from  './module' 
//等价于
import {es6 } form './module'
export default es6

模块的整体加载

用星号*指代一个对象，所有输出值都加载在这个对象上 

//module1.js
export var firstName = 'Michael';
export function func(){};  
//module2.js
import * as obj from './module1';
obj.func();
//调用module1的方法func()
obj.firstName
//'Michael'

module命令

module obj from './module1'
//等价于
import * as obj from './module1';

module后面跟一个变量，表示输入的模块定义在该变量上

export default

为了可以使用户不用查看模块源码去寻找模块暴露的接口，使用export default为模块指定默认输出

ES6笔记之三函数的扩展

函数参数的默认值

1、ES6允许为函数的参数设置默认值，即直接写在参数定义的后面 

function a(x = 3,y = 4){
    console.log(x);
    console.log(y);
}
a(); //3 , 4
a(2,4)  // 2 , 4

2、结合解构赋值设置默认值 

function foo({x,y  = 3}){  //参数是一个对象，只有传入的参数也是一个对象时，才会进行解构
    console.log(x);
    console.log(y);
}
foo({})                  // x = undefined , y = 3
foo({x = 4 , y = 1})     //x = 4 , y = 1 
-----------------------------------------------------------------------------------

function bar([x = 2,y] = [ 3, 4]){  //[3 , 4] 是为参数变量[ x = 2, y]设置的默认值
    console.log(x);
    console.log(y);
}
bar();        //x = 3 , y = 4
bar([3,45])   //x = 3 , y = 45
-----------------------------------------------------------------------------------

function bar([x = 2,y]){  //参数变量[ x = 2, y]没有设置默认值
    console.log(x);
    console.log(y);
}
bar();        //报错：VM425:2 Uncaught TypeError: Cannot read property 
              // 'Symbol(Symbol.iterator)' of undefined(…)
bar([3,45])   //x = 3 , y = 45

ES6笔记之二‘解构赋值’

定义

解构：ES6允许按照一定模式，从数组和对象中提取值，对变量进行赋值。

一、数组的结构赋值

ES6允许下面的写法：

上面的代码的作用是，从右边的数组中取值，按照位置对应的关系对左边的数组变量赋值。
只要两边的模式相同，左边的变量就会被赋予对应的值。

下面是解构成功的写法举例：

  • let [foo, [[bar], baz]] = [1, [[2], 3]]
    //foo=1, bar=2, baz = 3
  • let [ , , third] = [‘foo’,’bar’,’baz’]
    //third = ‘baz’
  • let [ head, …tail] = [1,2,3,4]
    //head=1, tail = [2,3,4]
  • let [x, y, z] = new Set([‘a’,’b’,’c’])
    //set结构赋值

下面是解构不成功的写法举例

  • var [foo] = [];
    //undefined
  • let [x,y, …z] = [‘a’]
    //x=a, y=undefined, z=[]
    左边数组共有三个子项，而右边只有一个，故只能给左边匹配的第一项赋值，余下的
    y与…z因为没有匹配项则值为undefined和[ ]

下面是解构不完全(但是解构成功)的写法举例

  • let [x, y ] = [1,3,3]
    //x = 1, y = 3
  • let [a,[b],d] = [1,[2,3],4]
    //a = 1, b = 2, d = 4

下面是解构出错的写法举例

  • let [x = y , y = 1] = [ ]
    //ReferenceError 因为x用到默认值y时y还没有声明

注意！ 如果等号的右边不是数组（即可遍历的结构），赋值将会出错。换言之，只要赋值的数据结构具有
Iterator接口，都可以采用数组形式的解构赋值 

//报错  
let [foo] = 1 / false / NaN /undefined / null / {}

结构赋值指定默认值

var [foo = true] = []  
// 若是未指定默认值的时候(var [foo] = [] )，其值将是undefined，但这里foo = true  
[x, y = 'b'] = ['a']  
// x = 'a', y = 'b'  
[3, y = 'b'] = ['a', undefined ]  
//x = 'a', y = 'b'  
let [x = f()] = [1]  
//默认值是表达式。该表达式遵循惰性求值，即只有在用到的时候才执行求值，而这里恰恰没有
//执行到，因为右边有赋值[1], x的结果将为 1.

这里应注意的是，ES6内部使用严格相等运算符“===”判断一个位置是否有值，故若不准备为某个变量赋值时
必须将其对应位置上的值设为‘undefined’！

二、对象的解构赋值

注意：
数组与对象的解构不同：数组的元素是按次序排列的，变量的取值由它的位置来定；而对象的属性没有次序，
变量必须与属性同名，才能取到正确的值。
(1)

var {foo, bar} = { bar : 'bbb', foo : 'aaa'};  
// 注意，左边和右边foo和bar出现的顺序并不一致，然而却能赋值成功
// foo = 'aaa' , bar = 'bbb'

在上例中，{ foo : ‘bbb’, bar : ‘aaa’ }，即相当于{x : a, y : b }。其中的 x 和 y 为对象的属性(即模式，永远不会变的)，而 a 与 b 则相当于变量(因为其值可以是任意的)。如果要赋值，则形式必须要符合
以下格式才有效： 

{ x : 23 , y : 'zhangsan'}

因为无论是数组还是对象的解构赋值的声明都相当于是在声明变量，故对let和const而言(var则不会)，一旦赋值的变量以前声明过，就会报错： 

let foo;  
let {foo } = { foo : 1}  //SyntaxError: redeclaration of let foo

(2)
和数组一样，嵌套结构的对象也能进行解构 

var obj = {
    p : [
        "hello",
       { y : 'world'}
    ]
}
var {p : [x,{y}]} = obj;   // x = 'hello'  
                           // y = 'world'

规则: 对象内，冒号“ ：”左边的符号就是模式，即永远不变的属性。
(3)
指定默认值
默认值生效的条件是，对象的属性值严格等于undefined 

var { x = 3 } = { }    
//  x = 3  
var { x , y = 5} = { x = 2 }  
//  x = 2 , y = 5

解构失败时，变量的值为undefined 

var  { foo } = { far : 200}  
foo = undefined

字符串的解构赋值

原理： 字符串在进行解构赋值前会先被转换成一个类数组的对象。 

const [a,b,c] = 'string';  
// a = 's'  b = 't' c = 'r'

数值和布尔值的解构赋值

如果等号右边是数值或布尔值，则会先转为对象 

let { toString : s} = 123;  
s === Number.prototype.toString  //true  

let { toString : s} = true ;  
s === Boolean.prototype.toString //true  

## 函数参数的解构赋值  
**(1)**
```javascript  
function add([x,y]){  //[x,y]实际上不是一个数组
    return x+ y ;
}

(2)
使用默认值的函数参数
1、

function move({x = 0 , y = 0} = {}){   //参数等号的右边为'{}'，作用是为函数的参数设置默认值。结果是即使是
    return [x,y];                        //像这样调用函数‘move()’也不会报错。这样赋值的形式类似于move( x = 0)
}                                      //其中x是参数变量(这里是“{x = 0 , y = 0 }”),而 0 是值(这里是“{ }”)
move({x :3 , y : 8 }) //[3,8]
move({x :3  })        //[3,0]
move({})              //[0,0]
move()                //[0,0]

上面的代码中，函数的参数是一个对象，并且这个对象内的属性都已经有默认值，若调用的时候没有新的值传入，则使用默认值。
2、 

function move({x,y} = {x:0,y:0}){
    return  [x,y];
}
move({x:3,y:8}) //[3,8] 
move({x:3})     //[3,undefined]
move({})        //[undefined,undefined]
move()          //[0,0]

上面的代码中，函数的参数也是一个对象，但其内部的属性并没有默认值，若调用的时候没有对应的值传入，则相应的值为undefined。
例1和例2两种写法的区别：
例1中函数参数的默认值是空对象，但是设置了对象解构赋值的默认值；
例2中函数参数的默认值是一个有具体属性的函数，但是没有设置对象解构赋值的默认值
